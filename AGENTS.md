# Agent Instructions

## Local Fork Changes

This fork maintains a curated list of fork-specific work in `README.md` under
`## Local Fork Changes`.

When making code or documentation changes, decide before finishing whether that
section should be updated. Update it when the change is durable and relevant to
someone using or maintaining this fork, including:

- User-visible behavior changes.
- Transcription preview or final transcription behavior changes.
- Provider-specific fixes, compatibility changes, or billing-impacting changes.
- Local build, install, signing, or developer workflow additions.
- Fork-maintained architecture or policy decisions.

Do not update the section for:

- Internal-only refactors.
- Test-only changes that do not document new behavior.
- Formatting, comments, or metadata-only changes.
- Temporary debugging changes.
- Build artifacts, caches, or ignored local files.
- Upstream-only merges that do not add fork-specific behavior.

Keep entries concise and product-oriented. Prefer one bullet per durable feature
or behavior, not one bullet per commit. When you update this section, mention it
in the final response.
